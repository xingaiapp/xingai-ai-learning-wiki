# XingAI Course Standard

This contract applies to every course and future lesson.

## Required Structure

1. **What:** definition, vocabulary, system boundary, and falsifiable claims.
2. **Why:** user problem, business outcome, and technical reason.
3. **Who:** users, builders, owners, reviewers, approvers, and affected parties.
4. **When:** selection signals, prerequisites, and when not to use the pattern.
5. **Where:** position in the request, data, control, trust, and organization boundaries.
6. **How:** implementation sequence, operations, validation, and retirement.
7. **Code:** minimal runnable example, expected behavior, extension exercise, and tests.
8. **Diagram:** Mermaid flow, sequence, state, data, trust, or decision view.
9. **Failure analysis:** quality, security, privacy, reliability, cost, and human-impact risks.
10. **Practice:** lab, interview ladder, rubric, and completion gate.

## Four-Depth Explanation

Each core concept must be explainable at four depths:

- Beginner: concrete mental model and vocabulary.
- Engineer: implementation, tests, and debugging.
- Architect: boundaries, alternatives, and non-functional requirements.
- CTO: business timing, portfolio fit, operating model, risk, and economics.

## Bilingual Parity

English `name.md` and Chinese `name.zh.md` must preserve the same outcomes, section order, code behavior, diagram meaning, exercises, references, and scoring. Chinese is a professional localization, not an abbreviated translation. Technical identifiers remain stable.

## Evidence Standard

Claims should be labeled as durable principle, versioned standard, vendor behavior, or XingAI design rule. Prefer primary sources: specifications, RFCs, research papers, official documentation, and internal ADRs. Record a verification date for version-sensitive material.

## Definition Of Done

- All 5W + How sections are substantive.
- Code is syntactically valid and has an explicit test path.
- Mermaid syntax is valid and communicates a real boundary or sequence.
- Risks include mitigations and residual risk.
- Interview questions include evidence expected at multiple levels.
- English and Chinese versions pass parity checks.
- Links resolve and vendor-sensitive claims are current at publication time.
