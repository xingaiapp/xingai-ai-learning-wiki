# XingAI AI 工程学习系列

从第一性原理到 CTO 级 AI 领导力。本双语课程要求学习者能够解释、构建、调试、运营、架构并领导 AI 系统，也用于准备常见 **AI 工程岗位面试流程**，但不承诺特定录用结果。

这是一个**文档课程**，不是 [Learn AI](https://learn.xingai.app)（[`xingai-learn`](https://github.com/xingaiapp/xingai-learn)）。Learn AI 根据用户提交的编程面试题推荐下一步 **Coding Pattern**；本课程通过课时与评估教授 **AI 工程师 / 架构师岗位能力**（系统设计、工具、RAG、MCP、治理）——不是刷题式 DSA Pattern 训练。

## 强制课程契约

每门课程均遵循 [COURSE-STANDARD.zh.md](COURSE-STANDARD.zh.md)：5W + How、代码、Mermaid 图、生产故障、实验、面试题和评分门槛。英文 `.md` 与中文 `.zh.md` 是内容完整、结构对应的两个版本。

## 学习方法

1. 阅读前先完成每门课的诊断题。
2. 不复制答案，独立完成代码练习。
3. 分别以初学者、工程师、架构师和高管深度讲解架构图。
4. 完成实验，并在作品集仓库中保存证据。
5. 达到 80% 课程门槛后再进入下一阶段。
6. 用新的资历层次回看早期课程；领导力需要技术判断，而不只是术语。

## 项目能力门槛

| 门槛 | 证明方式 |
|---|---|
| 理解 | 准确解释概念、假设与边界 |
| 实现 | 提交有测试、有类型边界的工作代码 |
| 调试 | 隔离模型、数据、工具、状态、安全和运行时故障 |
| 设计 | 比较替代方案，并用证据答辩权衡 |
| 领导 | 将架构与用户、风险、成本、团队和结果连接起来 |

## 高级路径

完成岗位相关基础课程后，继续学习[企业 AI 深度工程课程](../deep-enterprise-ai/README.zh.md)。该路径增加 Production-shaped 参考 POC、企业 RAG、Harness 与 Loop Engineering、Agent Framework 对比、受控多 Agent、MCP、身份/策略、安全、可观测性、审计、评估、运营与 CTO 架构答辩。
